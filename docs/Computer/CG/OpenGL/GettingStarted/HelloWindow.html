# 你好，窗口

新建一个存放源码的目录

mkdir src
cd src

新建一个main.cpp文件

写入如下代码

#include <glad/glad.h>
#include <GLFW/glfw3.h>

int main()
{
    glfwInit();
    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 3);
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3);
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_CORE_PROFILE);
    //glfwWindowHint(GLFW_OPENGL_FORWARD_COMPAT, GL_TRUE);

    return 0;
}

CAUTION

GLAD必须包含在GLFW之前，因为如果GLFW包含在前他会去系统找 OpenGL 头文件（通常是老版本），在这之后在包含GLAD就会触发重定义问题。

在main函数中调用glfwInit函数来初始化GLFW，然后使用glfwWindowHint函数来配置GLFW，

第一个参数代表选项的名称，我们可以从很多以GLFW_开头的枚举值中选择；第二个参数接受一个整型，用来设置这个选项的值。

上述代码中表达的意思是使用OpenGL3.3，并且使用OpenGL核心模式。

# 创建一个窗口对象

GLFWwindow* window = glfwCreateWindow(800, 600, "LearnOpenGL", NULL, NULL);
if (window == NULL)
{
    std::cout << "Failed to create GLFW window" << std::endl;
    glfwTerminate();
    return -1;
}
glfwMakeContextCurrent(window);

glfwCreateWindow()前两个参数是窗口的宽高，第三个窗口是窗口标题，最后两个参数暂时忽略。这个函数将会返回一个GLFWwindow对象。

如果窗口创建失败，使用glfwTerminate()终止GLFW程序。

如果创建成功就使用glfwMakeContextCurrent(window)将这个窗口作为GLFW上下文。

# 使用GLAD加载OpenGL函数指针

在调用任何OpenGL函数之前需要使用GLAD加载OpenGL的函数指针，对此我们初始化GLAD

if (!gladLoadGLLoader((GLADloadproc)glfwGetProcAddress))
{
    std::cout << "Failed to initialize GLAD" << std::endl;
    return -1;
}

glfwGetProcAddress这是GLFW提供的一个函数，他的作用是去系统中查找OpenGL函数的内存地址

GLADloadproc将glfwGetProcAddress返回的指针类型强制转换为GLADloadproc。

gladLoadGLLoader()将会利用GLFW返回的指针GLADloadproc，遍历所有 OpenGL 函数名，并将它们逐一映射到GLAD可以调用的函数指针上

# 视口

在开始渲染前，我们必须告诉OpenGL渲染区域的尺寸大小即视口(Viewport)，这样OpenGL才能知道怎么根据窗口的大小渲染图形，如果视口设置错误那么渲染出来的图形可能会发生形变。

glViewport(0, 0, 800, 600);

glViewport()函数前两个参数控制窗口左下角的位置。第三个和第四个参数控制渲染窗口的宽度和高度（像素）。

这时如果窗口的大小发生改变，那么你的viewport的大小也应该改变，所以我们可以注册一个GLFW提供的回调函数来响应这种变化

TIP

有些萌新可能不清楚回调函数是什么，简单来说通过规定固定的函数签名在固定的位置调用函数，但将实现函数逻辑的任务交给其他人

或者更加具体的：在既定的流程节点调用标准接口，而将接口内部的具体逻辑延迟至应用层去实现

GLFW提供了一个签名为

void (GLFWwindow* window, int width, int height);

的回调函数

void framebuffer_size_callback(GLFWwindow* window, int width, int height)
{
    glViewport(0, 0, width, height);
}

我们实现一个名为framebuffer_size_callback与签名一致的函数。

将其传递给

glfwSetFramebufferSizeCallback(window, framebuffer_size_callback);

这样当窗口的大小发生改变的时候就会调用framebuffer_size_callback函数，从而改变视口大小。

# 输入

我们希望程序能够处理用户的输入，GLFW可以很方便的完成这些内容。我们可以通过glfwGetKey来判断按键是否被按下

void processInput(GLFWwindow *window)
{
    if(glfwGetKey(window, GLFW_KEY_ESCAPE) == GLFW_PRESS)
        glfwSetWindowShouldClose(window, true);
}

# 准备好你的引擎

我们不希望只进行了一次渲染后应用程序就关闭了。我们希望在我们主动关闭程序之前不断的渲染，并且能够接受用户的输入。

因此我们可以添加一个while循环，我们称之为渲染循环

while(!glfwWindowShouldClose(window))
{
    processInput(window);
    
    glfwSwapBuffers(window);
    glfwPollEvents();    
}
  • glfwWindowShouldClose函数在我们每次循环的开始前检查一次GLFW是否被要求退出，如果是的话，该函数返回true，渲染循环将停止运行，之后我们就可以关闭应用程序。
  • glfwPollEvents函数检查有没有触发什么事件（比如键盘输入、鼠标移动等）、更新窗口状态，并调用对应的回调函数（可以通过回调方法手动设置）。
  • glfwSwapBuffers函数会交换颜色缓冲，颜色缓冲其实就是一个内存空间，存放的是下一张准备呈现在GLFW窗口的颜色数据，通常我们不会直接将渲染的结果直接呈现在GLFW窗口上，而是先渲染在缓冲区，在缓冲区准备好后再交换到GLFW窗口上

# 渲染

为了测试一切都正常工作，我们使用一个自定义的颜色清空屏幕。每个新的渲染迭代开始的时候我们总是希望清屏，否则我们仍能看见上一次迭代的渲染结果（这可能是你想要的效果，但通常这不是）。

我们可以使用glClear函数来清空屏幕的颜色缓冲，它接受一个缓冲位(Buffer Bit)来指定要清空的缓冲，可能的缓冲位有GL_COLOR_BUFFER_BIT（颜色缓冲），GL_DEPTH_BUFFER_BIT（深度缓冲）和GL_STENCIL_BUFFER_BIT（模板缓冲）。由于现在我们只关心颜色值，所以我们只清空颜色缓冲。

while(!glfwWindowShouldClose(window))
{
    processInput(window);
    
    glClearColor(0.2f, 0.3f, 0.3f, 1.0f);
	glClear(GL_COLOR_BUFFER_BIT);

    glfwSwapBuffers(window);
    glfwPollEvents();    
}

# 最后一件事情

当GLFW程序结束之前我们需要正确释放GLFW之前分配的资源，所以我们需要在退出之前使用

glfwTerminate();
return 0;
