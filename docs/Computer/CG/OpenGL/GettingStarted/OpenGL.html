# OpenGL

LearnOpenGL中文

OpenGL不是一个库，也不是一个 API 而是一个由Khronos组织制定并维护的规范。

OpenGL规范了每个函数的定义，具体实现交给了显卡生产商，显卡生产商需要根据OpenGL规范实现具体的函数，这里可以找到OpenGL3.3的规范文档。

# 核心模式与立即渲染模式

在OpenGL3.2之前OpenGL是使用立即渲染模式（固定管线，程序员不能自定义管线），在立即渲染模式下绘制图形很简单，OpenGL对底层细节做了很大程度的抽象，与此同时灵活性也大大被限制住了。后期程序员需要更多的灵活性，所以从OpenGL3.2开始，规范文档开始废除立即渲染模式，开启了核心模式的时代。

核心模式允许程序员能自定义管线（顶点着色器，片段着色器等），更多的底层控制，与此同时复杂性也提升了不少。

# 状态机

OpenGL是一个状态机：通过一系列变量描述OpenGL此刻该如何运行。OpenGL的状态被称为上下文。我们通过改变状态机的变量来渲染。

假设我们想绘制一个三角形边框，而不是三角形面我们可以设置状态机，让其去绘制线框而不是面。

我们通常是使用状态设置函数去改变上下文状态。

# 对象

OpenGL是使用C语言写的，所以可以把对象看作一个C语言结构体。
